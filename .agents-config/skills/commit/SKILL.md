# Commit

Создай коммит с осмысленным описанием на русском языке.

## Шаги

1. Выполни `git status` и `git diff` чтобы понять, что изменилось
2. Посмотри `git log --oneline -5` для стиля предыдущих коммитов
3. Сформируй сообщение коммита:
   - Первая строка: тип + краткое описание (feat/fix/refactor/docs/chore)
   - Пустая строка
   - Список ключевых изменений (если их больше одного)
4. Добавь нужные файлы в staging (избегай `git add .`, добавляй конкретные файлы)
5. Создай коммит
6. Покажи результат `git status`

## Формат сообщения

```
тип: краткое описание на русском

- Изменение 1
- Изменение 2

```

## Типы коммитов

- `feat` — новая функциональность
- `fix` — исправление бага
- `refactor` — рефакторинг без изменения поведения
- `docs` — документация
- `chore` — настройка, зависимости, скрипты

## Важно

- НЕ пушь автоматически — только коммит
- Если нет изменений — сообщи об этом
- Спроси подтверждение перед коммитом, если изменения большие или неочевидные
